Why

会议不是缺少记录，而是缺少可以继续使用的上下文。

很多工具最后只留下一段录音或一份长转录。真正需要回看时，用户还是要自己找重点、补结论、追待办。

录了音，但没人想重听

一小时的录音很完整，但它不是适合会后快速同步的材料。

有转录，但结论仍然分散

长文本解决了“有没有记录”，却没有解决“如何快速理解”。

有待办，但容易丢在聊天里

会议行动项如果不能进入日常工作流，就很容易失去跟踪。

Workflow

从会前到会后，减少每一步整理成本。

MeetMemo 不是单点转写工具，而是一条围绕会议场景设计的工作流。

01 · BEFORE

会前准备上下文

加入项目背景、专有名词或补充资料，让后续纪要更懂这场会议的业务语境。

FAQ

下载前你可能想确认这些问题。

MeetMemo 适合哪些会议场景？

日常会议、站会、1on1、客户访谈、需求评审、招聘面试、复盘会等需要留存上下文和行动项的场景。

这是 Mac app 还是网页工具？

MeetMemo 是 macOS 应用，不是浏览器网页工具。当前支持 macOS 15.5 及以上。

语音识别是否必须联网？

首次下载模型需要网络；模型准备完成后，SenseVoice / Fun-ASR-Nano 等转录可在本机运行，无需 STT API Key。

我的会议数据会上传到项目方服务器吗？

不会。会议文件、摘要和模板保存在你的 Mac。生成纪要时，会把必要文本发送给你自己配置的 LLM 服务商。

是否支持自定义总结格式？

支持。你可以使用内置会议模板，也可以创建自己的提示词模板，适配不同会议类型。

这个项目是否开源？

项目源码公开，采用 PolyForm Noncommercial License 1.0.0。个人、学习、研究、教育等非商业目的可以使用。
