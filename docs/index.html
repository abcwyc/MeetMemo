Problem

会议结束了，真正麻烦的部分才刚开始。

录音、聊天记录、零散笔记和待办经常分散在不同地方。过几天再回头看，很难快速知道当时讨论了什么、决定了什么、下一步谁负责。

很多会议不是没有记录，而是没有留下可继续使用的结论。

MeetMemo 的目标不是生成一段漂亮摘要，而是把会议内容变成之后仍能查找、引用和跟进的工作资产。

Workflow

为真实会议流程设计。

好的会议工具应该在会前、会中、会后都减少阻力，而不是在会议结束后再制造新的整理工作。

会前

准备会议背景、专有名词和补充资料，让后续纪要理解当前业务语境，而不是只处理一段孤立文本。

会中

保留讨论原貌，减少边听边记。麦克风和系统音频双路记录，更贴近线上会议的真实场景。

会后

快速获得总结、决策点和行动项，把会议内容沉淀为之后还能搜索、回看和复用的资料。

FAQ

下载前你可能想确认这些问题。

MeetMemo 适合哪些会议场景？

适合日常会议、站会、1on1、客户访谈、需求评审、招聘面试、复盘会等需要留存上下文和行动项的场景。

这是 Mac app 还是网页工具？

MeetMemo 是 macOS 原生应用，不是浏览器网页工具。当前支持 macOS 15.5 及以上。

语音识别是否必须联网？

首次下载本地模型需要网络；模型准备完成后，SenseVoice / Fun-ASR-Nano 等转录可在本机运行，无需 STT API Key。

我的会议数据会上传到项目方服务器吗？

不会。会议文件、摘要和模板保存在你的 Mac。生成纪要时，会把必要文本发送给你自己配置的 LLM 服务商。

是否支持自定义总结格式？

支持。你可以使用内置会议模板，也可以创建自己的提示词模板，适配不同会议类型。

这个项目是否开源？

项目源码公开，采用 PolyForm Noncommercial License 1.0.0。个人、学习、研究、教育等非商业目的可以使用。
