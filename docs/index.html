Features

为真实会议流程设计，不只是录音转文字。

从会前资料、实时转录、说话人识别，到纪要模板、行动项提醒和系统级语音输入，MeetMemo 让会议记录变成可执行的工作流。

本地多引擎语音识别

内置 SenseVoice、Fun-ASR-Nano 与 macOS SpeechAnalyzer。模型下载后转录在本机运行，无需额外购买 STT API Key，也减少音频离开设备的风险。

双路音频录制

同时捕获麦克风和系统音频，区分“自己”和“会议中的其他人”，线上会议、播放器声音都能进入转录流程。

说话人识别

使用 SenseVoice 或 Fun-ASR-Nano 时，可进一步区分不同发言人，纪要中直接标注谁说了什么。

模板化 AI 纪要

内置标准会议、1on1、客户访谈、需求评审、招聘面试、站会等模板，也支持自定义提示词。

行动项落地

从纪要中提取待办任务，一键加入 macOS 系统「提醒事项」，让会议结论可跟踪。

系统级语音输入

通过快捷键在微信、浏览器、Cursor、备忘录等前台应用里语音转文字并自动插入光标。

How it works

四步开始记录你的第一场会议。

下载并安装

从 GitHub Releases 下载最新版 MeetMemo，拖入 Applications 后打开。

完成权限授权

授权麦克风、系统录音、提醒事项等权限。可在系统设置中随时调整。

准备识别模型

选择 SenseVoice、Fun-ASR-Nano 或 macOS SpeechAnalyzer，首次使用按引导下载模型。

录制并生成纪要

创建会议、开始录制，结束后选择模板生成 Markdown 纪要并提取行动项。

Privacy first

会议数据保存在你的 Mac。

MeetMemo 不提供云端账号，也不会把你的会议数据同步到项目方服务器。你可以自行选择 LLM 服务商，并按组织合规要求决定哪些文本可以发送给模型。

本地优先的数据边界

音频识别在本机处理；会议、摘要、模板以 JSON 文件保存在本机 Documents 目录；LLM 凭据保存在 macOS Keychain。
