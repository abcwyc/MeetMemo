## Project Overview

MeetMemo (MeetMemo) is a free, open-source macOS AI meeting notetaker. It captures mic + system audio, streams to the configured STT provider for live transcription, and generates structured notes via the configured LLM provider.

Two sub-projects:
- `MeetMemo/` ‚Äî Native Swift/SwiftUI macOS app (primary)
- `landing-page/` ‚Äî Next.js 15 marketing site

## Commands

### Native App (Xcode)

Build from command line (do **not** run the app after building):
```bash
xcodebuild -project MeetMemo.xcodeproj -scheme MeetMemo -configuration Debug build
```

Release build (requires `.env` with `DEVELOPER_ID`, `APPLE_ID`, `TEAM_ID`, `APP_PASSWORD`):
```bash
./scripts/build_release.sh
```

Version management:
```bash
./scripts/update_version.sh [major|minor|patch|build|custom]
./scripts/verify_codesigning.sh
```

### Landing Page

```bash
cd landing-page
npm run dev    # Dev server
npm run build  # Production build
```

## Architecture

### Native App ‚Äî MVVM Layers

**Models** (`MeetMemo/Models/`)
- `Meeting.swift` ‚Äî Core data model: transcript chunks + generated notes
- `NoteTemplate.swift` ‚Äî 7 built-in templates + custom templates
- `Settings.swift` ‚Äî Provider credentials, prompts, onboarding state

**Managers** (singletons, `MeetMemo/Managers/`)
- `AudioManager.swift` ‚Äî Opens dual WebSocket connections through `STTProviderFactory`; the current implementation uses Doubao STT for mic + system audio
- `RecordingSessionManager.swift` ‚Äî Coordinates active recording state across the app
- `LocalStorageManager.swift` ‚Äî JSON persistence to `~/Documents/Meetings/` and `~/Documents/Templates/`; uses atomic writes (temp file ‚Üí replace)
- `KeychainHelper.swift` ‚Äî Secure STT and LLM credential storage in macOS Keychain
- `DataMigrationManager.swift` ‚Äî Versioned migration support for JSON data

**Services** (`MeetMemo/Services/`)
- `NotesGenerator.swift` ‚Äî Builds template-aware prompts and streams note content through the configured `LLMProvider`
- `LLMClient.swift` ‚Äî Routes Anthropic URLs to the Messages API and all other URLs to OpenAI-compatible chat completions
- `APIKeyValidator.swift` ‚Äî Validates STT and LLM configs before allowing recording
- `ErrorHandler.swift` ‚Äî Centralized error handling

**ProcessTap** (`MeetMemo/ProcessTap/`)
Low-level Core Audio subsystem for tapping system/app audio without UI access. `AudioProcessController` monitors running audio apps; `ProcessTap` manages tap lifecycle.

**ViewModels** (`MeetMemo/ViewModels/`)
SwiftUI `@Published` observable objects: `MeetingListViewModel`, `MeetingViewModel`, `SettingsViewModel`, `TemplatesViewModel`.

**Views** (`MeetMemo/Views/`)
SwiftUI views wired to ViewModels. Entry point is `ContentView.swift`; app bootstrap in `MeetMemoApp.swift` (Sparkle + PostHog initialization).

### Key Data Flow

1. **Recording start** ‚Üí `RecordingSessionManager` ‚Üí `AudioManager` opens two WebSocket streams via the STT provider factory (one mic, one system audio; currently Doubao) ‚Üí transcript chunks arrive and are appended to the active `Meeting` with a 2-second debounce before saving
2. **Note generation** ‚Üí `NotesGenerator` reads transcript + selected `NoteTemplate` ‚Üí `LLMClient` selects Anthropic Messages or OpenAI-compatible chat completions from the configured base URL ‚Üí streams markdown notes back to `RenderedNotesView`
3. **Persistence** ‚Üí all meetings/templates stored as JSON files; provider credentials in Keychain

### Landing Page

Next.js App Router under `landing-page/app/`. Pages: home, `/download`, `/privacy`, `/terms`. UI built with Shadcn (Radix UI + Tailwind CSS). No backend ‚Äî purely static marketing pages.

## Important Constraints

- **Build without running**: When modifying the native app, build to verify compilation but do not launch the app.
- **Code signing**: The release binary requires a Developer ID Application certificate. Debug builds run unsigned locally.
- **Universal binary**: Release builds target both `arm64` and `x86_64`.
- **Conventional commits**: Follow conventional commit format (`feat:`, `fix:`, `chore:`, etc.) per `CONTRIBUTING.md`.
